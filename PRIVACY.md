# Privacy Policy for Q1View

**Last updated: 2026-05-25**

## Summary

Q1View does not collect, transmit, or store any personal data. It is a local
desktop application that processes files you open on your own device.

## Data Collection

Q1View collects no data of any kind. Specifically:

- No usage analytics or telemetry are gathered.
- No crash reports are sent to any server.
- No personally identifiable information is collected or stored.
- No user accounts or registration are required.

## Network Access

Q1View does not make network connections during normal use. It reads image,
video, and raw frame files from the local file system and does not upload,
download, or transmit file content to any remote service.

## File Access

Q1View reads only the files you explicitly open through the file dialog, drag
and drop, or command-line arguments. It does not scan or index your file system.
Temporary in-memory copies of decoded frames are discarded when you close the
source or exit the application.

## Clipboard

When you use the paste (Ctrl+V) or capture (Ctrl+C) features, Q1View reads from
or writes to the Windows clipboard. This data stays on your device and is not
transmitted elsewhere.

## Third-Party Libraries

Q1View uses OpenCV/FFmpeg for video decoding and libheif for HEIF/AVIF decoding.
These libraries operate locally and do not make network connections.

## Children's Privacy

Q1View is a developer tool with no features directed at children. It does not
knowingly collect information from anyone.

## Changes to This Policy

If this policy changes in a meaningful way, the updated version will be
committed to this repository and the "Last updated" date above will be revised.
